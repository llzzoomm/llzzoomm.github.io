Java基础

2.equals的作用

  • Object类中默认的是== ，但是会重写

  • String类中比较字符串是否相同，区分大小写。

4.什么是面向对象

  • 面向对象相对于面向过程，是两种不同的处理问题方式
  • 面向过程，关注一件事情的步骤和顺序，比较直接高效高效
  • 面向对象关注一件事情的所有参与者（对象），各自做什么，注重可复用，维护，拓展。

5.1 封装

  • 封装的意义，明确标识出允许外部使用的成员函数和数据源内部细节对外部调用透明
  • 外部调用无需修改或者关心内部实现

5.2 继承

  • 子类继承父类的方法，子类和父类共性的方法或属性，可以直接用父类的，不需要自己定义，只需要个性化扩展

5.3 多态

  • 基于对象所属类的不同，外部对同一个方法的调用，实际执行的逻辑不同
  • 继承，方法重写，父类引用指向子类对象，不可以调用子类特有的方法

  • JDK java开发工具
  • JRE Java运行时环境
  • JVM java虚拟机
  • JDK包括JRE 和java工具
  • JRE包含bin(JVM)、lib(类库)

7.hashCode()和equals()

  • hashCode()的作用是获取哈希码，也成为散列码

  • 哈希码的作用是在哈希表中确定具体的位置

  • hashCode()定义在Object类里，所有的类都含有hashCode()

  • 哈希表存储的是键值对对象，可以根据哈希码快速的确定位置

  • 根据hashCode()讲述为什么hashSet里面存储的对象是不重复的

    • hash会根据hashCode()获取的哈希码，找到该对象的位置，然后判断这个位置是否有值，没有值则插入
    • 如果有值则调用equals方法判断这两个值是否相同，如果相同则该对象插入失败，如果不同则重新散列到其他位置
    • 这样减少了equals的比较次数，提高了执行效率

  • hashCode()的默认行为是对堆上的对象产生独特值。如果没有重写hashCode()，则该class的两个 对象无论如何都不会相等（即使这两个对象指向相同的数据）

8.final

  • 修饰类表示这个类不可以被继承
  • 修饰方法该方法不能被重写，但能重载
  • 修饰变量被赋值后就不能再被修改

（1）修饰成员变量

  • final修饰的类变量，只能在静态代码块或者声明是指定初始值（final static int a = 0）
  • final修饰的成员变量，只能在非静态代码块或者声明时或者构造器中指定初始值（final int b = 0）

（2）修饰局部变量

  • 局部变量系统不会指定默认值，只可以由程序员指定默认值，声明时不一定需要赋值，但是只能被赋值一次

（3）修饰基本类型和引用类型

  • 基本类型被赋值后不可修改
  • 引用类型被赋值后便不能再指向其他的对象，但是引用的值是可以改变的

（4）为什么局部内部类和匿名内部类只能访问final修饰的局部变量

  • 首先内部类和外部类的是同一个级别，外部类的生命周期结束不代表内部类的生命周期结束
  • 内部类只有当没有对象使用时才会结束
  • 当内部类调用局部变量时，如果外部类已经销毁那么局部类也没了，所以将局部变量复制了一份作为内部类对象
  • 但是这个时候这两个变量的值不一定相同
  • 所有就需要使用final修饰，用来确保内部类调用的变量是一致的

9.String、StringBuffer、StringBuilder

  • String是不可变的，由final修饰，每次操作都会产生一个新的String对象
  • StringBuffer是线程安全的由synchronized修饰
  • StringBuilder线程不安全
  • 性能StringBuilder>StringBuiffer>String

10.重载和重写的区别

  • 重载发生在同一个类中，方法名必须相同，参数的顺序、数量、类型不同，返回值和访问修饰符可以不同，发生在编译时
  • 重写发生在父子类中，方法名、参数列表必须相同，返回值的范围小于或等于父类，抛出异常范围小于或等于父类，修饰符大于或等于父类

11.接口和实现类的区别

  • 抽象类可以含有普通方法，而接口只能有public abstract方法
  • 接口可以多实现，而继承只能继承一个
  • 抽象类中的成员变量可以是多种类型的，而接口中只能是public static final类型

接口的设计目的是约束类的行为，可以强制不同的类具有不同的行为，但是如何实现可以不一样

抽象类的设计目的是提高代码的复用，父类和子类通用的方法可以继承，不一样的方法就单独实现

使用场景：当你关注一个事物的本质时用抽象类，关注操作时用接口

抽象类的功能远超接口，但是定义抽象类的代价高因为一个类只能继承一个类，那么就得将父子类的共识全部写出，

虽然接口的功能较弱，但是是针对动作的描述，而且一个类可多实现接口

12.List和set的区别

  • List可重复，有序，根据进入的顺序保存，可以含有多个null对象，可以使用Iterator取出，再逐一遍历，也可以使用get(index)取出
  • set不可重复，无序，只能含有一个null对象，只能使用Iterator取出，再逐一遍历.

13.ArrayList和linkedList的区别

  • ArrayList基于动态数组，存储在连续的内存，适合下标访问（随机访问），扩容机制：因为数组长度固定，超出长度需要重新创建数组，将老数组拷贝到新数组中，如果不是尾部插入数组还会使得都数组都移动，使用尾插法并指定初始容量可以极大地节省性能
  • LinkedList基于链表存储，可以存储在分散的内存，适合做数据插入和删除，不适合查询：需要逐一遍历
  • 遍历LinkedList必须使用Iterator不能使用for因为for每次都会重新遍历,极大地消耗性能
  • 不要尝试使用indexOf等返回元素索引，并利用其进行遍历，使用indexOf遍历list,当结果为空时会重新遍历整个列表

14.hashMap和hashTable的区别，底层实现原理是什么

区别：

  • hashMap没有synchronized修饰，线程不安全，hashTable线程安全
  • hashMap允许key和value为null,hashTable不允许

底层实现：

  • 数组＋链表实现
  • jdk8开始链表高度大于8，数组长度大于64自动转变为红黑树，元素以内部类node结点存在
  • 长度低于6将红黑树转回列表

15.ConcurrentHashMap简介

  • ConcurrentHashMap性对于hashMap是线程安全的，在多线程高并发的情况下常常用到
  • JDK1.8的实现已经抛弃了Segment分段锁机制，利用CAS+Synchronized来保证并发更新的安全，采用的数据结构（数组+链表+[红黑树]）

16.什么是字节码，使用字节码的好处是什么

Java中的解释器和编译器

  • Java中引入了虚拟机的概念，既在程序编译之后会生成字节码文件，该字节码文件供虚拟机访问，然后虚拟机解释执行该字节码文件转变成目标机器能识别的机器码，这也就解释了Java解释和编译并存的特点

字节码的好处

  • 在一定程度上解决了传统解释型语言执行效率低的问题，同时又保留了解释型语言易移植性的特点
  • 所以java语言运行效率高，只要编译了字节码文件，到其他目标机器上运行，不需要重新编译就可以运行

17.Java中的异常体系

  • Java中所有的异常都来源于顶级父类Throwable
  • Throwable有两个子类分别是Exception和Error
  • Error是程序无法解决的错误，一旦发生程序就会终止
  • Exception不要导致程序终止，又分为RunTimeException运行时异常，会导致当前线程执行失败。CheckedException检查时异常发生在编译阶段会导致编译不通过

18.Java类加载器

JDK自带三个类加载器分别是BootStrapClassLoader、ExtClassLoader、AppClassLoader

  • BootStrapClassLoader是ExtClassLoader的父类加载%JAVA_HOME%/bin下的jar包和class文件
  • ExtClassLoader是AppClassLoader的父类加载%JAVA_HOME%/bin/ext下的jar包和class文件
  • AppClassLoader是自定义加载器的父类，负责加载classPath下的类文件
  • 继承ClassLoader实现自定义加载器类

19.双亲委托模型

向上查找缓存，向下查找加载路径

好处

  • 主要是为了安全性，避免自己编写的类覆盖掉核心类比如：String
  • 也防止了类的重复记载，因为JVM区分不同的类，不仅仅是根据类名，相同的class文件由不同的类加载器加载就是不同的类

20.GC如何判断对象可以被回收

有两种方式

  • 引用计数法，每个对象有一个引用计数属性，新增引用+1，释放引用-1，当为0时回收
  • 可达性分析法：GC Roots开始向下搜索，搜索所走过的路径称为引用链，当没有引用链时判断对象是可回收的
